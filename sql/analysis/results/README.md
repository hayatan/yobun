# 評価結果ディレクトリ

このディレクトリには、スコアメソッド評価の結果を保存します。

## ディレクトリ構造

```
results/
├── README.md           # このファイル
└── YYYY-MM-DD/         # 評価実行日ごとのディレクトリ
    ├── evaluation_results.md  # 詳細レポート（全メソッド・全閾値の比較表）
    └── summary.md             # サマリー（クイックリファレンス）
```

## 評価結果の作成手順

### 1. 評価クエリの実行

`tolove_recommendation_evaluation.sql` をBigQueryで実行し、結果をCSVでエクスポートします。

### 2. 分析スクリプトの実行

```bash
cd sql/analysis
python3 scripts/analyze_batch_results.py /path/to/exported.csv > results/YYYY-MM-DD/evaluation_results.md
```

### 3. サマリーの作成

詳細レポートを元に、`summary.md` を作成・更新します。

## サマリーの記載フォーマット

### 必須項目

1. **クイックリファレンス表**: 全機種の比較表（TOP1/98%/99%の最高メソッドと機械割）
2. **機種別詳細分析**: 各機種について以下を記載
   - 特徴（1行）
   - 評価期間別の推奨候補テーブル
   - 🎯 推奨メソッド（優先順位付き、理由付き）
   - 📊 信頼度評価（5段階星評価 + 詳細）

### 信頼度評価の基準

| 評価 | 星 | 勝率の目安 | 機械割の目安 | 狙い定めの効果 |
|------|-----|-----------|-------------|---------------|
| 非常に高い | ⭐⭐⭐⭐⭐ | 60%超 | 108%超 | ◎ 極めて有効 |
| 高い | ⭐⭐⭐⭐ | 55〜60% | 105〜108% | ○ 有効 |
| 中程度 | ⭐⭐⭐ | 50〜55% | 102〜105% | △〜○ 普通 |
| 低い | ⭐⭐ | 45〜50% | 100〜102% | △ 限定的 |
| 非常に低い | ⭐ | 45%未満 | 100%未満 | ✗ 無効 |

### 勝率・機械割の評価基準

- **勝率60%超**: 成績が良い
- **勝率65%超**: かなり良い
- **機械割105%後半**: 成績が良い
- **機械割108%超**: かなり良い
- **機械割100%未満**: 期待値マイナス（狙わない方が良い）

### 信頼度評価に含める項目

```markdown
**📊 信頼度評価**: ⭐⭐⭐⭐⭐（非常に高い）
- **TOPいくつまで信頼可能**: 勝率○%・機械割○%
- **THRESHOLDいくつまで推奨**: ○%でも機械割○%を維持（平均○台）
- **狙い定めの効果**: ◎ 極めて有効。コメント
```

### 推奨メソッドの記載例

```markdown
**🎯 推奨メソッド**:
1. **`original`**（最推奨）- TOP1も98%/99%も安定して高く、複数台狙いに最適
2. **`rms_frequency`** - 120日TOP1で110.49%と最高だが、98%/99%では`original`に劣る
```

## 注意事項

- 評価結果は過去データに基づくもので、将来の成績を保証するものではありません
- 特日設定を変更した場合は再評価が必要です
- 定期的に評価を更新し、傾向の変化を確認することを推奨します
