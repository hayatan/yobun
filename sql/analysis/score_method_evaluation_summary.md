# スコアメソッド評価結果サマリー

## 📋 概要

10機種（2店舗）について、8種類のスコアメソッドを評価し、各機種の最適なメソッドを特定しました。

**評価期間**: 直近120日間  
**評価日数**: 70日（機種により若干の差異あり）  
**評価方法**: 過去データのみを使用したバックテスト（未来の情報を使用しない）

## 📈 傾向分析

### 店舗別の傾向

| 店舗 | `original` | `rms_frequency_dual` | `rms_frequency_filter` | `rms_reliability` |
|------|-----------|---------------------|----------------------|-------------------|
| **アイランド秋葉原店** | **5機種** | 1機種 | 0機種 | 0機種 |
| **エスパス秋葉原駅前店** | 0機種 | 0機種 | **3機種** | **3機種** |

**発見事項**:
- **アイランド秋葉原店**: `original` メソッドが圧倒的に有効（5機種中5機種）
- **エスパス秋葉原駅前店**: `rms_frequency_filter` と `rms_reliability` が有効（6機種中3機種ずつ）

### 機種別の傾向

| 機種 | アイランド | エスパス |
|------|-----------|---------|
| **L+ToLOVEるダークネス** | `rms_frequency_dual` | `rms_frequency_filter` |
| **Lソードアート・オンライン** | `original` | `rms_frequency_filter` |
| **Lバンドリ！** | `original` | `rms_reliability` |
| **L戦国乙女４** | `original` | `rms_reliability` |
| **スマスロ+マギアレコード** | `original` | `rms_frequency_filter` |

**発見事項**:
- 同じ機種でも店舗によって最適メソッドが異なる
- 店舗の特性（設置数、設定投入のクセなど）がメソッドの有効性に影響

### 店舗別の設定投入パターンの仮説

#### エスパス秋葉原駅前店の特徴
- **観察**: `rms_frequency_filter` と `rms_reliability` が有効
- **仮説**: **特定の番号の台を軸に並びで設定を入れる傾向がある**
  - 例: 末尾1桁が同じ台、末尾2桁が同じ台など、特定の数字パターンに沿って設定投入
  - この傾向により、特定の台番号パターンが複数の戦略で重複して選ばれる
  - `rms_frequency_filter` は低品質な戦略を除外し、高品質な戦略のみで頻度をカウントするため、このパターンに適している
  - `rms_reliability` はデータ量を重視するため、頻繁に出現するパターンが評価されやすい

#### アイランド秋葉原店の特徴
- **観察**: `original` メソッドが圧倒的に有効
- **仮説**: **より複雑な条件の組み合わせで設定投入を行う傾向がある**
  - 単純な台番号パターンだけでなく、勝率・機械割・差枚ランキングなど複数の条件を組み合わせて判断
  - `original` メソッドは全要素（信頼性、頻度、異常値、複合条件）を考慮するため、複雑な条件にも対応できる

---

## 🎯 スコアメソッドの説明

### `original`
- **計算式**: RMS × 信頼性スコア × 頻度ボーナス × 異常値ボーナス × 複合ボーナス
- **特徴**: 最も複雑な計算。全要素を考慮。
- **有効性**: アイランド秋葉原店で圧倒的に有効
- **適用場面**: 複数の条件（勝率・機械割・差枚ランキングなど）を組み合わせて設定投入する傾向がある店舗に適している。全要素を考慮するため、複雑な条件の組み合わせにも対応できる。

### `rms_frequency_dual`
- **計算式**: RMS × 頻度ボーナス × 複合ボーナス
- **特徴**: 頻度と複合条件を重視。シンプルながら効果的。
- **有効性**: アイランドのL+ToLOVEるダークネスで最高パフォーマンス

### `rms_frequency_filter`
- **計算式**: RMS × 頻度ボーナス（有効性スコア0.5以上の戦略のみ使用）
- **特徴**: 低品質な戦略をフィルタリング。精度重視。
- **有効性**: エスパス秋葉原駅前店で3機種で有効
- **適用場面**: 特定の台番号パターン（末尾1桁、末尾2桁など）を軸に設定投入する傾向がある店舗に適している。高品質な戦略のみで頻度をカウントするため、パターン性の高い設定投入に反応しやすい。

### `rms_reliability`
- **計算式**: RMS × 信頼性スコア（絶対閾値ベース）
- **特徴**: データ量と参照回数を重視。安定性重視。
- **有効性**: エスパス秋葉原駅前店で3機種で有効
- **適用場面**: 特定の台番号パターンが頻繁に出現する店舗に適している。データ量と参照回数を重視するため、繰り返し出現するパターンが高く評価される。

## 📝 評価データの詳細

### 評価期間
- **開始日**: 2025-11-03以降のデータ
- **評価日数**: 直近120日間（機種により70日程度）
- **評価方法**: 各評価日について、その日の前日データのみを使用して推奨台を算出し、実際のパフォーマンスを記録

### 評価指標
- **勝率**: 推奨台に座った日のうち、差枚がプラスだった日の割合
- **機械割**: 推奨台の平均機械割（差枚+ゲーム数×3）/（ゲーム数×3）
- **平均差枚**: 推奨台の平均差枚

### 評価カテゴリ
- **TOP1**: スコア1位の台のみ
- **TOP2-5**: スコア上位2-5台
- **OUTLIER**: 飛び抜けて高いスコアの台（Q3 + 1.0*IQR または 平均 + 1.5*標準偏差）
- **THRESHOLD_80-99%**: TOP1スコアの各%以上の台

## 🤖 AI用プロンプト（他の店舗・機種での分析再現用）

以下のプロンプトをコピーしてAIに貼り付けることで、新しい店舗・機種でも同様の分析を再現できます。

### 評価クエリ実行プロンプト

```
# スコアメソッド評価タスク

## 目的
指定された店舗・機種について、複数のスコアメソッドを評価し、最適なメソッドを特定します。

## 前提条件
- 評価クエリ: `sql/analysis/tolove_recommendation_evaluation.sql`
- データマート: `yobun-450512.datamart.machine_stats`

## 実行手順

### Step 1: 評価クエリのパラメータ設定
`tolove_recommendation_evaluation.sql` の `params` CTEを以下のように変更してください：

```sql
WITH params AS (
  SELECT
    '[店舗名]' AS target_hole,      -- 対象店舗
    '[機種名]' AS target_machine,   -- 対象機種
    120 AS evaluation_days,         -- 評価期間（推奨: 120日以上）
    -- 特日タイプ:
    --   'island' = アイランド秋葉原店（6,16,26日、月末）
    --   'espas'  = エスパス秋葉原駅前店（6,16,26日、14日、月末）
    '[特日タイプ]' AS special_day_type
),
```

**特日タイプの選択**:
- **アイランド秋葉原店**: `'island'`（0のつく日、1のつく日、6のつく日、月末）
- **エスパス秋葉原駅前店**: `'espas'`（月末のみ）
- **特日なし**: `'none'`（全日を通常日として扱う）
- **その他**: 店舗の特日を調べて、適切なタイプを選択または新規追加

**注意**: 特日タイプを指定すると、戦略条件に「特日」「特日以外」が追加されます。
これにより、特日の傾向を検出できます。特日を考慮しない場合は `'none'` を指定してください。

### Step 2: BigQueryで実行
上記のパラメータを設定したクエリをBigQueryで実行し、結果をコピーしてください。

### Step 3: 結果の分析
結果データを貼り付けて、以下の分析を依頼してください：

「このデータから以下を分析してください：
1. TOP1で最も成績が良いメソッドはどれか（勝率・機械割・平均差枚で判断）
2. 複数台に絞る場合（THRESHOLD_95-99%）で最も成績が良いメソッドはどれか
3. TOP1も悪くなく、複数台でも良いメソッドはどれか
4. 推奨閾値（1日平均1-2台程度に収まる閾値）」

### Step 4: 結果のドキュメント化
分析結果を `score_method_evaluation_summary.md` に追加してください。

## 注意事項
- 機種名は完全一致で指定してください（部分一致不可）
- 店舗名も完全一致で指定してください
- 評価期間は最低60日以上、推奨120日以上
```
